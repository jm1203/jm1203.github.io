# Jimin's Tech Blog: 포스팅 작성 가이드

이 문서는 p0lestar 테마 기반으로 커스텀된 현재 블로그에서 마크다운(Markdown) 포스팅을 작성할 때 참고할 수 있는 공식 가이드라인입니다. 나중에 글을 쓰실 때마다 이 파일을 열어보고 복사해서 사용하시면 편합니다!

## 1. 파일 생성 규칙

- **경로**: 반드시 `_posts/` 폴더 내부에 마크다운 파일을 생성합니다.
- **파일명**: `YYYY-MM-DD-원하는-제목.md` 형식을 엄격하게 지켜야 합니다.
  - 예시: `2026-06-05-aws-ocsf-integration.md`
  - 띄어쓰기 대신 하이픈(`-`)을 사용하는 것이 안전합니다.

## 2. Front Matter (문서 속성 템플릿)

문서의 최상단에는 반드시 `---` 로 둘러싸인 메타데이터(Front Matter)가 있어야 합니다. 아래 템플릿을 복사해서 새 글의 맨 위에 붙여넣기 하세요.

```yaml
---
layout: post
title: "포스팅의 실제 제목을 적어주세요"
date: 2026-06-05 20:00:00 +0900
category: Cloud
tags: [AWS, Security, OCSF]
description: "포스트 제목 아래에 출력될 소제목(요약 설명)입니다."
pin: false
hide_toc: false
---
```

### 💡 주요 속성 팁
- `category` (대분류): 카테고리는 가급적 1개만 지정하는 것이 좋습니다. (메인 화면 필터 칩에 렌더링됨)
- `tags` (소분류): 대괄호 `[]` 안에 쉼표로 구분하여 여러 개를 넣을 수 있습니다.
- `pin` (고정글): `true`로 설정하면 메인 화면 최상단에 항상 고정되어 노출됩니다. 핵심 포트폴리오 글에 활용하세요.
- `hide_toc` (목차 숨김): 이 테마는 우측에 목차(Table of Contents)를 자동 생성해 줍니다. 글이 너무 짧아서 목차가 필요 없다면 `true`로 설정하세요.

## 3. 핵심 마크다운 문법

### 목차(TOC) 생성 기준
우측 목차는 `## (Heading 2)`와 `### (Heading 3)` 태그를 기준으로 자동 생성됩니다.

```markdown
## 1. 아키텍처 개요
내용을 작성합니다.

### 1.1. 상세 분석
내용을 작성합니다.
```

### 코드 블록 (Syntax Highlighting)
개발 블로그의 꽃인 코드 하이라이팅은 백틱(`) 3개와 언어 이름을 명시하여 사용합니다.

````markdown
```python
def analyze_cloud_log():
    print("Multi-Cloud Log Integration")
```
````

### 인용문 및 하이라이트 박스
```markdown
> 이 기호는 중요한 문장이나 인용구를 눈에 띄게 감싸줄 때 사용합니다. 다크 모드에서 아주 예쁘게 렌더링됩니다.
```

### 이미지 삽입
로컬 이미지를 띄우려면 `assets/img/` 폴더를 활용하세요. 
(예를 들어 `assets/img/posts/2026-06/` 처럼 월별로 폴더를 나눠서 이미지를 관리하면 아주 깔끔합니다.)
```markdown
![클라우드 아키텍처 다이어그램](/assets/img/posts/2026-06/architecture.png)
```

## 4. 커밋 및 배포 (Husky & Commitlint 규칙)

현재 지민님의 로컬 환경에는 엄격한 커밋 메시지 컨벤션(Husky)이 걸려있습니다. 포스팅을 다 작성하셨다면 아래와 같이 소문자로 시작하는 접두어(주로 `docs:` 활용)를 붙여서 배포해야 에러가 나지 않습니다.

```bash
# 1. 변경사항 추가
git add .

# 2. 커밋 (반드시 소문자 + 콜론 띄어쓰기 지킬 것)
git commit -m "docs: AWS OCSF 연동 관련 신규 포스팅 작성"

# 3. 배포
git push origin main
```

**[기타 유용한 커밋 태그]**
- `docs:` : 포스팅 글 작성 및 마크다운 수정
- `feat:` : 블로그 새로운 레이아웃, 기능 추가
- `fix:` : 오타 수정이나 버그 해결
- `style:` : CSS, 폰트 등 디자인 변경
- `chore:` : 설정 파일 등 기타 잡무
